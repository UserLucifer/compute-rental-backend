# 算力租赁后端系统测试用例 v1.0

生成日期：2026-05-10  
适用范围：当前后端项目、`业务手册.html` 已确认口径、Spring Boot 后端接口与定时任务。

## 1. 测试边界

### 1.1 本轮必须覆盖

- 用户注册、登录、忘记密码、JWT 鉴权。
- 商品目录、收益测算、租赁下单、机器费支付、API 模拟凭证、部署费支付、自动暂停、用户启动。
- 每日收益、佣金、到期结算、提前结算。
- 钱包、充值、提现、钱包流水和资金对账。
- 后台登录、运营配置、用户/订单/资金查询、充值提现审核、定时任务手动触发。
- 消息、推送设备、博客、文档等配套业务。
- 权限、安全、幂等、并发和异常输入。

### 1.2 本轮不作为失败项

- API Token 只作为页面模拟凭证展示，不交付可复制调用的明文 token。
- 商品库存、已租库存、可用库存和可租日期只做展示/运营参考，不参与下单交易校验。
- 充值不做链上自动确认，当前为人工审核入账。
- 提现不做链上自动打款回调，当前以管理员确认打款为准。
- 人工调账、人工结算审核、部署订单退款、业务事件自动通知属于未实现或下一期范围。
- 自动暂停由定时任务扫描触发，不保证秒级精确。

## 12. 每日收益、佣金、结算

## 19. 测试通过标准

- P0 用例全部通过。
- P1 用例无影响资金安全、鉴权安全、订单状态一致性的失败。
- 钱包金额不允许出现负数、重复入账、重复扣款、累计字段与流水不一致。
- 租赁订单、API 凭证、部署订单、收益记录、结算单、佣金记录之间状态必须一致。
- 后台审核类操作必须有状态防重和操作日志。
- 定时任务必须有执行日志，并可通过后台手动触发验证。
- 所有失败响应必须是统一错误结构，不能泄露堆栈或 SQL。
